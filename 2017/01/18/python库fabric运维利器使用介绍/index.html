完整示例

应用场景一：
发布jee应用程序到tomcat服务器，手动通过eclipse打包程序为xxx.war,然后通过fabric发布
1、获取svn版本号
2、从本地复制war包到远程服务器，在指定release目录解压，目录名称为当前日期+svn版本好的目录
3、删除tomcat的webapps下原来的符号链接，建立新的符号链接
4、回滚到先前的指定版本
5、重启tomcat

应用场景二：
同时更新2台nginx的web页面
1、获取svn版本号
2、从svn库打包页面并上传到服务器，在指定release目录解压，目录名称为当前日期+svn版本好的目录
3、web1上删除站点目录下之前的符号链接，并建立新的符号链接。web2上不操作。
4、回滚到先前的指定版本
