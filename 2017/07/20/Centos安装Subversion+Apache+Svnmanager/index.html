2.3、创建svn版本库

即创建/opt/svndata目录，此目录为svn的根目录，并在此目录下创建了myproject版本库。
因为我们最终是要让subversion支持多个版本库，因此关于svn账号、授权的信息不能放在某个单独的库下，因此我们直接放在/opt/svndata这个svn的版本库根目录下，主要是三个文件：
authz：用于存放版本库的授权信息。
passwd：用于存放账号和密码。
svnserve.conf：用于svnserve形式启动的配置文件。

2.4、创建svn账号

为安全起见，将用户密码作处理后保存到文件中，这就需要使用Apache自带的htpasswd工具，它可以生成passwd文件，其中用户名为明文，用户密码为md5哈希值。

如果没有htpasswd命令，安装httpd-tools包。
如果指定的目录不存在passwd文件，就需要带上”-c”参数，表示创建文件，但是之后就不需要了。如果每次都带上，反而只有最后一次的执行结果。

三、Subversion和Apache集成

上面就可以直接使用svn协议来操作，下面我们在进行和apache的集成配置，使svn库可以通过http协议来进行访问。
使用http来操作svn就不需要通过svnserve来启动svn服务了

3.1、设置httpd加载svn相关模块

subverion编译时会将生成的mod_dav_svn.so及mod_authz_svn.so拷贝到/usr/local/httpd-2.4.25/modules目录.

3.3、权限设置

因为Apache以apache/apache这个user/group来运行，因此需要让apache账号有读写/opt/svndata目录下的权限，可执行命令：
chown –R apache /opt/svndata
