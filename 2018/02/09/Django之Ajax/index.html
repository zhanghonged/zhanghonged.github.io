Django之Ajax

发表于 2018-02-09 更新于 2018-02-10 分类于 Django

Ajax 局部提交:
我们发起一次请求，就要提交一次内容，当我们因为一点数据瞬间刷新整个页面的时候，我们意识到这是一种浪费，也是一种糟糕的用户体验。所以我们在提交数据的时候，只把部分需要的数据提交，然后刷新页面部分。
Ajax 刷新局部，我们需要用js来实现。

Ajax 只是使用 js 实现局部提交的一种技术,区别在用后端用什么来响应:
Python下可用Django、Cgi、Flask等，
也可用php、java等。
这节内容我们使用Django作为后端响应来实现几个小例子。

Ajax用法介绍

url:&nbsp;&nbsp;&nbsp;&nbsp;提交的地址
type:&nbsp;&nbsp;&nbsp;&nbsp;提交的类型
data :&nbsp;&nbsp;&nbsp;&nbsp;提交的数据

  • get提交的数据写在链接上，get请求目的是向服务器请求数据(分页数据，普通页面刷新等)，
    • 以**?**问号开始
    • 键**=**值的形式
    • 以**&**分割
  • post主要是用来提交数据(注册，登录等)
    • 请求的数据需要以 字典 或者 json 的格式发送
    • 在视图函数当中需要用 render 进行返回
    • 请求需要携带 csrf

success:&nbsp;&nbsp;&nbsp;&nbsp;如果成功调用的函数,函数一定要有一个参数来接收返回的值
error:&nbsp;&nbsp;&nbsp;&nbsp;如果请求失败调用的函数,函数一定要有一个参数来接收返回的值
