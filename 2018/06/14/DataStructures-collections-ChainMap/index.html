DataStructures-collections-ChainMap

发表于 2018-06-14 更新于 2018-06-15 分类于 PyMOTW-3

collections：容器数据类型。
collections 模块实现了专业的容器数据类型用来替代Python的通用内置容器，字典, 列表, 集合, 和 元组。

collections 包括以下方法:

  • ChainMap — 为多个映射创建单一视图的类字典类型
  • Counter — dict子类，用于计算可哈希对象的个数
  • defaultdict — 调用一个工厂函数来为dict的values缺失提供值
  • deque — 双向队列,一个能在“队列”两端快速出队、入队的，类似于队列的（list-like）的容器
  • namedtuple — 这是个工厂方法,生成有命名字段的tuple子类
  • OrderedDict — 有序字典，dict子类 记录着数据成员添加的顺序
  • collections.abc — 容器的抽象基类

ChainMap：搜索多个字典。
ChainMap 类管理一个字典序列，并按照给定的顺序对它们进行搜索，以查找与键相关的值。
class collections.*ChainMap(maps)
一个 ChainMap 组合多个字典或其它映射集创建一个单一且可更新的视图。如果没有指定 maps，那么将会用一个空字典作为新 chain 里面的一个 mapping。
基础的 mappings 被存储在一个列表当中。这个列表是公开的，可以用 maps 属性访问和更新。没有其它的情况。

1、取值

ChainMap 支持与常规字典相同的API来取值。

子映射按照传递给构造函数的顺序进行搜索，因此键 c 的值来自于字典 a。

2、重新排序

ChainMap 在其 maps 属性中存储搜索的映射列表。这个列表是可变的，因此可以添加新的 mapping 或更改元素的顺序以控制查找和更新行为。

当 mapping 反序后，c 的值就改变了。

3、更新值

ChainMap 并不缓存子映射中的值，因此他们的内容更改后访问 ChainMap 也会发生改变。

修改元素和添加新元素效果相同。

4、new_child

new_child(m=None)
返回一个新的 ChainMap，它包含一个新的 map 和当前实例的所有 maps，如果指定 m 参数，那么它会成为新 maps 列表里面的第一个 map ; 如果没有被给,那么将会使用空字典作为第一个 map ，所以调用 m1.new_child() 等价于: **ChainMap({}, *m1.maps)**。
此方法用于创建子空间，当子空间的值被更新将不会影响父 mappings 里面的任何值。
