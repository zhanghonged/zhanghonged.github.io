Zabbix-短信告警

发表于 2018-06-27 | 分类于 Zabbix

为了能够更时效的发布告警到指定用户，可通过发短信的方式来通知指定用户。
短信告警 也是通过 脚本 类型的报警媒介来实现。

配置方面与 邮件告警 相比有以下不同：
1、短信告警需要用到短信平台，是收费的，短信模板需要提交平台审核。
2、短信告警内容没有标题，只有内容。
下面我们详细介绍配置方法。

一、短信平台配置

首选选择一款短信平台，亿互无线、云片网等，我这里介绍用的是亿互无线。

2、配置模板

故障【变量】,确认人:【变量】,确认时间:【变量】,消息:【变量】.主机:【变量】,故障:【变量】.

2、脚本编写 mobile.py

三、zabbix web端配置

1、增加报警媒介类型

  • 进入 管理 -> 媒介类型
  • 点击 创建媒介类型

类型： 选择脚本
脚本名称： 填写上一步创建的脚本 mobile.py
脚本参数： 向脚本添加命令行参数，是zabbix的 宏 变量

{ALERT.SENDTO} 收件手机号
{ALERT.MESSAGE} 内容

2、为用户分配报警提示

  • 进入 管理 -> 用户
  • 打开用户属性窗体
  • 在 报警媒介 选项卡 点击 添加

类型： 选择上一步创建的媒介类型 短信报警
收件人： 填写收件人手机号码
如果存在严重性则使用 标记要激活警示标记的触发严重性的复选框

3、配置报警动作

  • 进入 配置 -> 动作
  • 右上角的 事件源 下拉框选择 触发器
  • 点击 创建动作

注意：这里和配置邮件报警的不同，邮件报警可以把  标题  和  内容  直接给 SMTP 服务器去发送，但短信报警不同，必须去匹配平台的短信模板。

计算式： 多个条件的表达式，默认AND与不同的条件类型和OR具有相同的条件类型
条件： 动作触发的条件
