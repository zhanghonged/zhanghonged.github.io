Svn提交自动构建Maven项目

发表于 2017-01-06 更新于 2017-02-08 分类于 日常运维

架构

一般项目开发流程为开发人员提交代码到版本库、编译打包、部署到服务器，最后测试人员开始测试。这个流程牵扯到开发、运维、测试相关人员，颇为繁琐。
为了简化此流程我们采用如下架构：
svn提交代码——->jenkins打包—–> jenkins发布
1、开发人员提交svn代码到版本库。
2、提交动作触发钩子脚本。
3、钩子脚本远程执行jenkins打包项目。
4、jenkins持续集成完成打包、发布等动作。

使用Xmlrpclib操作cobbler_api

发表于 2016-11-08 更新于 2017-02-08 分类于 日常运维

介绍

XMLRPC是一个用不同语言编写的通过网络进行交互的轻量级计算机程序。
如果你想在cobbler上实现以下目的，可以使用XMLRPC API：

  • 你想跟Cobbler进行交互，但是你的应用不是python的程序/脚本；
  • 你想跟Cobbler进行交互，但是你的程序没有运行在Cobbler服务器上；
  • 你有一个non-GPLd应用程序，并且想跟Cobbler交互，并被分发给公众或其他的客户。

官方地址：https://fedorahosted.org/cobbler/wiki/CobblerXmlrpc

cobbler介绍

Centos7安装Saltstack

发表于 2016-10-19 更新于 2018-03-01 分类于 Saltstack

SaltStack是一个服务器基础架构集中化管理平台，具备配置管理、远程执行、监控等功能，一般可以理解为简化版的puppet和加强版的func。SaltStack基于Python语言实现，结合轻量级消息队列（ZeroMQ）与Python第三方模块（Pyzmq、PyCrypto、Pyjinjia2、python-msgpack和PyYAML等）构建。

通过部署SaltStack环境，我们可以在成千上万台服务器上做到批量执行命令，根据不同业务特性进行配置集中化管理、分发文件、采集服务器数据、操作系统基础及软件包管理等，SaltStack是运维人员提高工作效率、规范业务配置与操作的利器。
